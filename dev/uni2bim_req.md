你的任务是：**在不破坏现有 BimanGrasp 行为的前提下，只做最小必要修改，增加一个 `uni2bim` 模式，实现左右手独立优化。**

代码实现过程中你可以使用bimangrasp这个conda环境进行代码调试，你写的代码要用英文

请严格遵守：

* 不重构整体框架。
* 不修改默认模式（无 `uni2bim` 时行为必须完全一致）。
* 只在指定分支下做局部改动。

具体要求如下。

---

### 1. 增加一个配置开关

1. 在现有配置或入口脚本中增加：

   * `mode: uni2bim`（或等价布尔开关 `use_uni2bim`）。
2. 所有后续特殊逻辑仅在 `mode == "uni2bim"` 时生效。

**要求：**如果不启用 `uni2bim`，代码路径与当前版本完全一致。

---

### 2. 在 `uni2bim` 模式中关闭跨手耦合项（用权重=0 实现）

在当前组合 loss 的位置（负责加权 `E_dis / E_fc / E_vew / E_objpen / E_selfpen / E_bimpen / E_joint` 的地方）：

当 `mode == "uni2bim"` 时，仅做下面这些修改：

1. 将 **左右手互穿项** 关闭：

   * 对 `E_bimpen`（或等价 inter-hand penetration 项）对应的权重设为 0，不参与总能量。
2. 将 **协同稳定性项** 关闭（仅当依赖两手联合信息时）：

   * 对 `E_vew`（或任何基于两手共同 G/接触点的 wrench 体积或全局稳定性项）对应权重设为 0。
3. 若存在专门的「双手联合 force closure」项（如 `bimanual_fc`, `global_fc` 这类显式用左右手接触点一起构造的项）：

   * 在 `uni2bim` 模式下将其权重设为 0。

**不要**改这些项的实现函数本身，只通过条件分支控制是否加到总 loss 里。

---

### 3. 唯一必须改公式结构的地方：force closure 拆成左右手两份

如果当前 `E_fc` 的实现是**用左右手接触点一起构造单个 grasp matrix / G**，则在 `uni2bim` 模式下做如下**最小修改**：

1. 在现有 `E_fc` 计算函数或其调用处，增加一个 `if mode == "uni2bim"` 分支：

   * 用已有代码分别对**左手接触点**构造 `G_left`，得到 `E_fc_left`；
   * 对**右手接触点**构造 `G_right`，得到 `E_fc_right`；
   * 定义：
     [
     E_fc_\text{uni2bim} = E_fc_\text{left} + E_fc_\text{right}
     ]
2. 在 `mode == "uni2bim"` 时，总能量中使用 `E_fc_uni2bim`，而**不要**使用原本那种「合并左右手接触点」的 `E_fc`。
3. 在非 `uni2bim` 模式下，保持原有 `E_fc` 逻辑不变。

这是唯一需要动公式结构的地方，请将改动局限在这一处。

---

### 4. 其它能量项的处理（只用现有实现的按手加和，不新造东西）

在 `uni2bim` 模式下，其它项只做“按手独立 + 求和”的解释，不必大改代码：

* `E_dis`：如果当前实现已按左右手分别累加，可直接保留；否则在实现中区分左右手点，各自累加后求和。
* `E_objpen` / 物体穿模：使用现有对每只手与物体的检测逻辑，相加即可。
* `E_selfpen`：保留仅检查**单手内部**自碰的部分；任何左右手之间的碰撞检查已在第 2 步通过权重=0 关闭。
* `E_joint`：保留现有对每只手关节越界的惩罚。

**不要**引入新的复杂项，尽量复用已有实现，只通过条件判断决定是否包含跨手部分。

---

### 5. 核心性质检查

完成修改后，`mode == "uni2bim"` 时应满足：

1. 总能量形如：
   [
   E_{\text{uni2bim}} =
   E^{(L)}*{\text{local}} + E^{(R)}*{\text{local}}
   ]
   仅包含对每只手自身与物体、自身几何相关的项。
2. 模型反向传播时：

   * 左手参数的梯度只来源于左手相关项；
   * 右手参数只来源于右手相关项；
   * 不再有任何内部公式同时依赖左右手状态。
3. 默认模式（非 `uni2bim`）行为与现在完全一致。

请只做上述必要修改，避免无关重构或改名。
